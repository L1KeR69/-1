# Постановка задачи

## Актуальность
Портал дистанционного обучения нужен организациям, которые хотят запускать онлайн-обучение без разработки LMS с нуля.

## Ограничения
- проект должен быть понятен как Open Source решение;
- демонстрационные данные должны быть обезличены;
- система должна поддерживать облачную подписку и самостоятельное развертывание;
- ключевые метрики должны быть связаны с модулями системы.

## Критерии успешности
1. Можно зарегистрировать организацию и пользователей.
2. Пользователь может пройти курс, а система фиксирует прогресс.
3. Администратор видит активность пользователей и организаций.
4. Можно анализировать ARR, Retention 30, конверсию в подписку и завершение курсов.

## Риски
- Сложный интерфейс снижает процент завершения курсов.
- Пользователи забывают возвращаться к обучению.
- Ошибки авторизации мешают регулярному использованию.

## Связь с GitHub Issue
Эта постановка задачи относится к Issue «Сформулировать проблему и цель LMS-портала» и фиксирует критерии успешности проекта через метрики ARR, Retention 30, конверсию в подписку и завершение курсов.
